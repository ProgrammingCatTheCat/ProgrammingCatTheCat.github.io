个人网站搭建教程（3）

这一章使我们个人网站搭建的最终章，我们现在就开始吧！

05 git本地库关联远程库

首先，我们要在ProgrammingCatTheCat/public中初始化git库。

之后网页的提交也是先在本地更改，本地文件中的public文件也自动更改，git上只提交public文件夹内容。

那么为了创建出public文件，我们需要先构建这个结构，代码为：

hugo -F --cleanDestinationDir

也可以只写一个hugo,但是带-F的那个是指public每次更改后都清空重建，更加’干净‘一些。

这样之后我们的文件中就有了public。

现在我们先进入public目录，然后输入git init 初始化Git库。

在ProgrammingCatTheCat/public目录下，为git本地库添加远程库：

git remote add origin git@github.com:ProgrammingCatTheCat/ProgrammingCatTheCat.github.io.git

然后查看config文件：

cat .git/config

如果返回值里有[remote “origin”]，那么说明本地库已经成功关联远程库啦！

06 提交修改

首先，我们需要把修改提交到本地库上。

代码如下：

git status #查看修改

git add . #添加所有修改

git commit -m "my first commit" #提交修改到本地库，commit备注为“my first commit”

然后，我们把本地的commi提交到远程库，代码为：

git push -u origin master

这个时候，我们可以在github的项目上看到推送的branch，代表我们的博客配置完成了！

不过别急，还有一步。

github创建的项目默认branch是main，但是我们提交的branch是master。

个人网站搭建教程（2）

上一篇我们安装了hugo，配置了github仓库，做好了个人网站搭建的准备。

这一篇我们将真正地搭建个人网站，在本篇结束时，你就已经拥有一个自己的网站了！

03 利用hugo新建网站和网页

首先我们在一个文件夹下新建一个hugo网站，这里我还是在E：\MyBlog中打开gitbash（或者终端），输入：

hugo new site 网站名

我想把我的网站名取名为ProgrammingCatTheCat-Blog，所以我的命令是

hugo new site ProgrammingCatTheCat-Blog

在建立完网站之后，我想要新建一篇网页。所以，我可以打开刚刚创建的site，使用如下命令新建一个markdown文件：

hugo new post/test.md，表示新建一个post文件夹，在文件夹中新建test.md文件。

这样我们就建立了一个ProgrammingCatTheCat的site，并且往其中塞进去一个文件test，很快就可以渲染可视化了。

04 选择hugo皮肤

hugo官方有提供很多皮肤，我们可以选择自己喜欢的皮肤用来搭建博客。

网址如下：Complete List | Hugo Themes (gohugo.io)

我的选择是beautiful hugo,就是这个：Beautiful Hugo | Hugo Themes (gohugo.io)

下载的时候在对应site文件夹的themes文件中下载，例如我的命令为：

git clone https://github.com/halogenica/beautifulhugo.git beautiful

具体格式为git clone + https地址 + 给它命名的新文件名。

原本的代码也可以去掉最后的beautiful，只写

git clone https://github.com/halogenica/beautifulhugo.git

但是这样的话文件名就是原本的名字，比较长，不方便。所以我下载后给它取了个新名字beautiful

下载好模板后，我们再返回site根目录，找到hugo.toml文件，添加一行内容： theme = "beautiful"

这里我填beautiful，是因为我下载的模板名字叫beautiful，如果你下载的是别的模板，就需要自己修改一下theme的主题啦。

接下来我们写一行代码，看一看效果：

hugo server

如果顺利的话，终端会跳出一大段话，然后让你去localhost:1313预览网页结果。

好的，那么我们现在已经预览到了本地的静态网页，下一章就是最终章，部署网站到github托管了。

我们不见不散！

个人网站搭建教程（1）

本篇博客介绍了通过hugo和github搭建个人网站的过程。

在搭建博客的时候，我也遇到了很多坑，在此记录下来，希望可以对你有一点帮助。

1.hugo配置

Hugo是一个静态网站生成器。

更具体的说，它是一个开源的个人博客框架，运行速度快，有很多网站模板和皮肤可以选择。

首先我们肯定是要下载并且配置hugo，hugo的官网如下：

Releases · gohugoio/hugo (github.com)

下拉assets可以找到各种版本，选择适配自己电脑型号的版本就好，最好选包含extended的版本，因为之后hugo皮肤配置的时候，extended适配的版本要更多一点。

解压后放到合适的目录下，比如我就把hugo放到E盘的MyBlog文件中。

解压完成后要在电脑中配置系统环境设置，开始搜索框搜索“高级系统设置”，点击“环境变量”，在“系统变量”中找到path变量，点击“编辑”，选择“新建”，把hugo添加进去。

比如我的path变量就是 “E:\MyBlog\hugo_extended_0.123.7_windows-amd64”

添加环境变量是为了让计算机可以找到hugo的位置，添加完成后重启计算机， 再次打开hugo所在的文件夹（在我这里是E：\MyBlog，然后用cmd终端或者git bash打开，输入hugo version。

如果有类似的显示，就证明hugo安装成功，可以使用啦。

2.github仓库配置

安装了hugo之后，我们就可以利用它搭建一个博客框架了。

但是具体的内容（网站中展示的文件）还需要我们自己设置。为了方便起见，我们可以选择在github中建立一个项目，每次更改之后，把文件（网页）推送到这个项目里进行储藏。

要注意的是Repository Name取名应该为：github用户名.github.io

我的github用户名是ProgrammingCatTheCat，所以我取的项目名为：ProgrammingCatTheCat.github.io

所以在我的网页部署之后，它的url也就会是：https://programmingcatthecat.github.io/

项目建成之后我们可以先把它丢一边，接下来我们需要新建网站和网页，然后用hugo渲染它们。
